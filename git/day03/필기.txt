1. push와 pull
 1-1. 원격으로 커밋 밀어올리기 : psuh
 git push
    -> 이미 git push -u origin main 으로 대상 원격 브랜치가 지정되어 있기 때문에 git push 명령어만 해도 상관 x

 1-2. 원격의 커밋 당겨오기 : pull
  git pull
 1-3. pull 할것이 있을때 push를 한다면?
  - 원격에 먼저 적용된 새 버전이 있으므로 적용 불가
  - pull 해서 원격의 버전을 받아온 다음 push가능
  - push 를 하려면 원격 저장소 대로 맞춰져 있어야 가능하다.
 1-4. push할 것이 있을시 pull 하는 방법
  - git pull --no-rebase : merge방식
  - git pull --rebase : rebase 방식

 1-5. 협업상 충돌 발생
 1-6. 로컬의 내역 강제 push 해보기
  - 원격의 내용이 local보다 뒤쳐져 있으면 push를 할 수가 없다. 그러나 원격 내용이 잘못되서 local에 있는 내용으로 강제로 맞춰줘야 할 때가 있다. **협의가 반드시 필요한 상황
